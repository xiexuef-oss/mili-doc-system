# 必填章节与约束

## 1. 范围 [必填] [≥300字]
必须包含：产品名称（用实际值）、研制阶段（用实际值）、大纲适用范围和目的

## 2. 引用文件 [必填]
至少列出：GJB 450B-2021、GJB 451A-2005、GJB 899A-2009
以及上下文中列出的其他标准。格式：标准编号 标准名称

## 3. 术语和定义 [必填] [≥5条]
从 GJB 451A 引用核心术语（可靠性、MTBF、MTBCF、任务可靠性、基本可靠性等）

## 4. 可靠性要求 [必填] [≥500字]
必须包含：
- 定量要求：MTBF、MTBCF、R(t)、使用寿命（从已知数据取实际值，缺失的标记）
- 定性要求：故障判据、验证方法
- 任务剖面和环境条件（从已知数据取实际值）

## 5. 可靠性工作项目 [必填] [≥800字]
按 GJB 450B 表 1 列出各系列工作项目：
- 100系列（可靠性管理）：101-104
- 200系列（可靠性设计与分析）：201-203
- 300系列（专项）：301-304、310、311、313
- 400系列（试验与评价）：401-403
- 500系列（使用可靠性评估）
每项说明：目的、工作要点、实施时机、输出物

## 6. 可靠性验证 [必填] [≥300字]
包含：验证方法选择、统计试验方案、故障判据、接收/拒收准则

## 7. 可靠性管理 [必填] [≥200字]
包含：组织架构、各阶段工作安排、FRACAS、转承制方管理、评审计划

# 生成约束

1. **锁定已知值**：项目上下文"已知数据"中的所有数值和事实，原样使用，不得修改
2. **标记不确定**：不确定的内容用 `[⚠待核实: 具体说明缺什么]` 标记
3. **禁止占位符**：禁止使用"XXX"、"待补充"、"略"、"详见XX"、"TBD"
4. **每章输出 AI_META**：格式见下方
5. **输出完整 Markdown**：章用 ##，节用 ###
6. **不输出开场白和结束语**
